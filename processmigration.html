Dynamic Process Migration in Agent Based Simulation

With Dr. Dhananjai Rao, and Paul Bondurant

As a freshman at Miami in 2013, I worked with Dr. Dhananjai Rao and Paul Bondurant on dynamic process migration in agent based simulation. Agent based simulations track interactions between independent agents interacting in an environment. As with all simulations, all interactions are dependent on the clock. For the simulation to yield proper results, all interacting agents must be on the same clock time. This is not usually an issue in serial agent based simulations. In parallel agent based simulations, agents in one process may be at a different clock time than agents in another process. Even if the two processes started at the same time, minute hardware differences, discrepancies in required processing, and the random nature of simulations can all cause clocks between processes to be out of sync.

The time-warp algorithm fixes this issue. If two agents on different clock times interact, time is turned back for the agent furthest in the future. The issue with this algorithm is that it is very time consuming to run. When an agent rewinds, all of its interactions must be redone, which can take considerable processing time. The goal of this research, Paul Bondurant's Masters thesis, was to limit the the number of interactions by keeping groups of agents who interacted often on the same clock time. Using a multi-node compute cluster, Paul devised an algorithm to move commonly communicating processes to similar nodes, to cut down on time warps.

This was my first experience with novel research. To help Paul, I performed documentation and ran tests on Miami's Redhawk compute cluster. I learned far more than I bargained for about high performance computing, parallelism, simulation, and research in general. After helping on this project, I was hooked. I continued with research throughout undergrad with the Air Force, and now I am pursuing a PhD.
